# ISO 27001 Control Mapping

This document demonstrates how common security controls map to domains in the ISO 27001 framework.

The purpose of control mapping is to ensure that implemented security practices align with governance and compliance requirements.

---

## Example Control Mapping

| Security Control | ISO 27001 Domain | Description |
|------------------|------------------|-------------|
| Multi-Factor Authentication (MFA) | Access Control | Ensures only authorized users can access systems and sensitive data. |
| Encryption at Rest | Cryptography | Protects stored data from unauthorized access. |
| Encryption in Transit | Communications Security | Protects data transmitted across networks. |
| Logging and Monitoring | Operations Security | Detects suspicious or unauthorized system activity. |
| User Access Reviews | Access Control | Periodically verifies that users have appropriate access rights. |

---

## Why Control Mapping Matters

Organizations must demonstrate that their security controls align with recognized frameworks such as ISO 27001.  

Control mapping helps security teams:

- Identify compliance gaps
- Prepare for audits
- Ensure security policies align with industry standards

---

## Framework Reference

ISO 27001 includes a set of security domains (Annex A) covering areas such as:

- Access Control
- Cryptography
- Operations Security
- Communications Security
- Information Security Policies

These domains guide organizations in implementing effective security governance.
