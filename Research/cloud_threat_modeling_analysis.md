# Case Study: Cloud Security Threat Modeling & Risk Assessment

## Overview

This case study analyzes cybersecurity threats in cloud computing environments, focusing on threat modeling techniques and risk assessment methodologies.

Based on academic research conducted at Rutgers University.

---

## Key Security Threats Identified

- Data breaches
- Account hijacking
- Insider threats
- API vulnerabilities
- Malware / ransomware
- Misconfiguration

---

## Analysis of Security Controls

The following controls were evaluated:

- Encryption
- Firewalls
- Access Control
- Multi-Factor Authentication (MFA)
- Intrusion Detection Systems (IDS)

### Key Insight

Traditional controls provide baseline protection but are insufficient for dynamic cloud environments.

---

## Threat Modeling Techniques

- STRIDE
- PASTA
- LINDDUN
- Attack Trees
- OCTAVE

### Key Insight

STRIDE is widely used due to simplicity but depends heavily on user expertise and proper implementation.

---

## Risk Assessment Approach

- Qualitative risk analysis
- Fuzzy logic for risk scoring

### Key Insight

Fuzzy logic improves prioritization of cybersecurity risks under uncertainty.

---

## GRC Perspective

From a Governance, Risk, and Compliance perspective:

- Cloud environments require adaptive risk models
- Static frameworks struggle with evolving threats
- Proper documentation and structured processes are critical
- Human error significantly impacts risk outcomes

---

## Lessons Learned

- Threat modeling is only as strong as its implementation
- Improper use of frameworks leads to missed threats
- Cloud security requires continuous reassessment
- Combining methodologies improves effectiveness

## Real-World Application

If applied in an enterprise environment, this analysis would support:

- Cloud risk assessments
- Security control evaluations
- Vendor risk analysis
- Compliance readiness (ISO 27001 / NIST)
